# Plan 模板

生成 Plan 文档时使用以下结构。编号从 `plans/` 目录最大编号 +1 开始。

```markdown
# Plan {编号} — {标题}

> **Status**: Draft
> **Owner**: TBD
> **Created**: {YYYY-MM-DD}
> **Depends on**: {外部依赖或前置 Plan，无则写"无"}
> **Target area**: {涉及的主要目录/文件路径}

## 1. 背景

### 1.1 问题描述

{用户痛点、业务需求、或技术债务的描述}

### 1.2 现状审计

{当前代码/架构的现状，可用表格对比"当前实现"vs"目标状态"}

| 维度 | 当前实现 | 目标状态 |
|---|---|---|
| ... | ... | ... |

### 1.3 设计目标

{编号列出 3-5 条核心设计目标}

## 2. Goals

{编号列出本次 Plan 要达成的具体成果，每条可验证}

## 3. Non-Goals

{明确列出不在本次范围内、但可能被误解为在范围内的内容}

## 4. 关键决策

### 4.1 {决策标题}

{决策内容 + 决策理由 + 备选方案对比}

## 5. 实施计划

### Phase 1: {阶段名称}

**目标**: {Phase 要达成的效果，一句话}

**覆盖文件**:
- `path/to/file1.ts`
- `path/to/file2.tsx`

**任务**:
- [ ] 任务 1 描述
- [ ] 任务 2 描述
- [ ] 任务 3 描述

**Acceptance Criteria**:
1. {可验证的完成标准 1}
2. {可验证的完成标准 2}

<!-- 执行完成后回填：
**Reviewed**: <日期> by code-reviewer
**Commit**: <commit hash> <commit message 标题>
-->

---

### Phase 2: {阶段名称}

{同上结构}

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| ... | ... | ... |
```

## Test Cases 模板

Plan 全部 Phase 完成且 review 通过后，在文档末尾追加：

```markdown
## Test Cases

### TC-001: {用例标题}

- **前置**: {环境 / 数据 / 入口状态}
- **步骤**:
  1. {具体操作，点击/输入/命令}
  2. {具体操作}
  3. {具体操作}
- **路径**: `packages/.../foo.ts:L12`, `packages/renderer/src/pages/Bar.tsx`
- **预期**: {可观测的 UI、日志、DB 状态、事件}

### TC-002: {用例标题}

- **前置**: ...
- **步骤**:
  1. ...
  2. ...
- **路径**: ...
- **预期**: ...
```
